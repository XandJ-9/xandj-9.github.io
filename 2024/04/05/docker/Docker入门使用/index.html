1. Docker基本概念

容器（container）

原文介绍

​ Simply put, a container is a sandboxed process on your machine that is isolated from all other processes on the host machine. That isolation leverages kernel namespaces and cgroups, features that have been in Linux for a long time. Docker has worked to make these capabilities approachable and easy to use. To summarize, a container:

  • Is a runnable instance of an image. You can create, start, stop, move, or delete a container using the DockerAPI or CLI.
  • Can be run on local machines, virtual machines or deployed to the cloud.
  • Is portable (can be run on any OS).
  • Is isolated from other containers and runs its own software, binaries, and configurations.
  • 容器是镜像的实例，你可以通过Docker命令或对应的API接口来创建，启动，停止，移动，删除一个容器；
  • 容器可以运行在本地机器上，也可以在虚拟机或者远程的云服务器上运行；
  • 支持在仍和容器上运行；
  • 不同容器之间完全的物理隔离。

Docker中利用容器独立运行一个应用程序或一组应用，应用程序运行在容器里，容器就相当于在一个虚拟化的运行环境。

镜像（image）

​ 容器实例运行在一个独立的虚拟环境中，而这个虚拟环境则是有镜像创建，在镜像中定义好了容器运行需要的一系列依赖，以及环境变量的信息。

仓库（repository）

​ 仓库就是存放镜像的地方，自定义创建好的镜像可以推送到仓库中，方便在新的环境中迁移使用。使用仓库可以方便镜像的集中管理。

​ Docker本事是一个容器运行的载体或称为管理引擎， 应用程序和配置依赖打包好之后形成一个可交付的运行环境，这个打包好的运行环境就是镜像文件，只有通过这个镜像文件才能生成Docker容器实例（类似于Java中一个类new出一个对象实例）。

关于docker run 相关的命令行参数

命令格式：docker run [OPTIONS] IMAGE [COMMAND] [ARG…]
Usage: Run a command in a new container
中文意思为：通过run命令创建一个新的容器（container）

常用选项说明

示例
运行一个在后台执行的容器，同时，还能用控制台管理：

docker run -i -t -d ubuntu:latest

运行一个带命令在后台不断执行的容器，不直接展示容器内部信息：

docker run -d ubuntu:latest ping www.docker.com

运行一个在后台不断执行的容器，同时带有命令，程序被终止后还能重启继续跑，还能用控制台管理，

docker run -d –restart=always ubuntu:latest ping www.docker.com

为容器指定一个名字，

docker run -d –name=ubuntu_server ubuntu:latest

容器暴露80端口，并指定宿主机80端口与其通信(: 之前是宿主机端口，之后是容器需暴露的端口)，

docker run -d –name=ubuntu_server -p 80:80 ubuntu:latest

指定容器内目录与宿主机目录共享(: 之前是宿主机文件夹，之后是容器需共享的文件夹)，

docker run -d –name=ubuntu_server -v /etc/www:/var/www ubuntu:latest

3. 数据卷 Volume

数据卷是什么

​ 容器内目录与宿主机器目录的映射，方便重要数据文件的保存和共享。

匿名挂载

​ 在启动容器时直接使用-v /container_dir 即可完成匿名绑定，匿名绑定的方式将在Docker的volumes目录下生成一个sha256方式生成的字符串作为目录名，且指定的容器目录/container_dir下的文件也会同步出现在该字符串目录下，匿名指定的目录在容器被删除时，指定目录下的文件也会被删除。

​ 不指定宿主机的目录

具名挂载

​ 给容器挂载数据卷的同时，指定数据卷的名称，通过名字可以快速管理这些数据卷。
